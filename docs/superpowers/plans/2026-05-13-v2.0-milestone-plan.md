# DL-Processor v2.0 — Milestone Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task.

**Goal:** Ship v2.0 — de-iframe refactor + Business Process grouping + DLD drift detection — as a unified milestone distributed as the first patch-based update to v1.2.0 users.

**Architecture:** Three feature areas in one release. Feature A (de-iframe) is the foundation; Feature B (BP grouping) builds on it natively from day 1; Feature C (DLD drift) is an isolated backend refactor that extracts compare.js extractors into a reusable module.

## Phase 3 — Feature A (de-iframe refactor)

Reason for going BEFORE BP grouping: Feature B builds on the native renderer pattern. Doing A first means B doesn't need to "build in iframes then refactor" — saves rework.

### Task 4: Tab-host refactor — native `<div>` panes

**Files:**
- Modify: `electron/renderer/tab-host.js`
- Modify: `electron/renderer/app.js` (update call sites)

The current `tabHost.open({ url, srcdoc, title })` creates an iframe. New API: `tabHost.open({ title, render })` where `render(container)` is called with a `<div>` the tab-host owns. Existing url-based tabs (Dashboard, etc.) keep iframe support for external HTML files — only srcdoc tabs convert.

Steps:
- [ ] Read current tab-host.js end-to-end.
- [ ] Refactor: add `render` mode alongside existing url mode. Drop srcdoc mode entirely (no existing callers will be left after Phase 3).
- [ ] Update call sites in app.js that pass srcdoc (review-pending, history) — leave them broken for now; the next tasks fix them.
- [ ] No test changes (renderer not unit-tested).
- [ ] Manual smoke: dashboard tab still opens correctly via url mode.
- [ ] Commit: `refactor(tab-host): support render-mode tabs; drop srcdoc`
