## Phase 2 — Forward chain + user attribution

### Task 5: `writeAuditLog` chains forward + auto-fills user
- Modify `src/audit-log.js`:
  - Accept optional `user` field in the event object; if absent, call `currentUser()`
  - After INSERT, call `chainAppend(db, newRowId)` to compute + UPDATE prev_hash + row_hash
  - Wrap INSERT + UPDATE in `db.transaction()` so partial chains can't exist
- 4 tests: user auto-fills, hash chains to prior row, hash deterministic across runs, backfill+new-row chain is contiguous
- Target 403 → 407
