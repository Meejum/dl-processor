# v2.3 Workflow Automation — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship v2.3 — rules engine, anomaly badges, bulk operations, trending tile, pre-compare dry-run — as documented in `docs/superpowers/specs/2026-05-18-v2.3-workflow-automation-design.md`. Forward-only from v2.2.0.

**Architecture:** Single rules engine becomes the spine for every automated decision (built-ins + user rules). Built-in anomaly predicates also exist as pure functions for fast renderer-side filtering. Bulk ops + trending tile + dry-run hang off the same shared infrastructure (rule engine, audit_log, pending_change). One migration (009) creates the rule table, adds `pending_change.anomaly`, widens `audit_log.source` CHECK to enum (pattern validation moves to app layer because SQLite CHECK can't use LIKE).

**Tech Stack:** Node.js + better-sqlite3 + Electron + node:test. No new dependencies. Patch ships on top of v2.2.0.

**Test count target:** 428 → ~480 (+52).

**Phases:** 0 spikes → 1 schema + bootstrap audit → 2 rule engine → 3 anomaly → 4 wire compare-drift → 5 rule loader + built-ins + refactor auto-approve → 6 bulk ops → 7 trending → 8 dry-run → 9 rules editor UI → 10 anomaly UI in Review Pending → 11 settings + sidebar + dashboard tile → 12 integration + ship.

**In-flight branches:** none. Clean apply.

**Hot-file mitigation:** `compare.js` SF-branch edits are surgical (single wire point); `writeAuditLog` gets a non-breaking new optional param; `auto-approve.js` collapses to a thin wrapper around the rule engine.

  up(db) {

      CREATE TABLE IF NOT EXISTS automation_rule (
        id            INTEGER PRIMARY KEY,
        name          TEXT NOT NULL,
        enabled       INTEGER NOT NULL DEFAULT 1,
        priority      INTEGER NOT NULL,
        when_json     TEXT NOT NULL,
        then_json     TEXT NOT NULL,
        builtin       INTEGER NOT NULL DEFAULT 0,
        created_at    TEXT NOT NULL,
        created_by    TEXT,

        ON automation_rule(enabled, priority);
    `);

    // 2. pending_change.anomaly column (idempotent via PRAGMA check)
    const cols = db.prepare("PRAGMA table_info(pending_change)").all();
    if (!cols.some(c => c.name === 'anomaly')) {
      db.exec("ALTER TABLE pending_change ADD COLUMN anomaly TEXT");

    // Pattern validation (rule:<id> / bulk:<UUID>) lives in src/audit-log.js validateAuditSource()

        CREATE TABLE audit_log_new (
          audit_id     INTEGER PRIMARY KEY AUTOINCREMENT,
          ts           TEXT NOT NULL,
          table_name   TEXT NOT NULL,
          row_pk       INTEGER NOT NULL,
          field        TEXT NOT NULL,
          action       TEXT NOT NULL CHECK (action IN
            ('approve','reject','override','auto_apply','learn_alias',
             'approve_bp','reject_bp','acknowledge_bp','revert')),

          user         TEXT,
          user_note    TEXT,

          prev_hash    TEXT,
          row_hash     TEXT

        DROP TABLE audit_log;
