# DL-Processor v2.2 — Native Project Dashboard

**Status:** Approved by Ali — 2026-05-14
**Branch:** `feat/v2.2-native-dashboard` (off `master` at `3be7ffe`)
**Target test count:** 422 → ~428

## Problem

v2.1 has two surfaces that still render as `<iframe src="file://…">` against static HTML files written by the CLI:

- `output/dashboard.html` — portfolio overview, one card per project.
- `output/compare/<slug>.compare.html` — per-project DLD vs SF table.

These work, but they live in a different context from the rest of the app:

1. They can't dispatch the `dlp:open-history` event we added in v1.1, so clicking a procedure number is dead-end.
2. They can't open the per-unit history side panel (`unit-history-panel.js`), so users can't pivot from a compare row to its audit trail.
3. They can't call `window.dlp` IPC bridges, so a `PENDING` chip can't deep-link to Review Pending.
4. They can't read v2.1 audit hardening additions (user, hash chain, Tier-2 flag) — the static HTML is frozen at compare-run time.

v2.2 makes both pages first-class renderer-DOM tabs while leaving the static HTML files in place for offline distribution.

## Goals

1. Replace the iframe Dashboard with a native render-mode tab.
2. Replace the iframe Project Compare with a native render-mode tab.
3. Click a unit row → open per-unit history side panel.
4. Click a procedure number → open History tab filtered by that procedure.
5. Click a `PENDING` chip → open Review Pending tab filtered to that unit.
6. CLI `compare` keeps writing the static HTML files unchanged.

## Non-goals

- Re-importing DLD/SF data from inside the Project Compare page (CLI `[1]/[2] Import` is still the path).
- Editing `master_data` from the compare table (Review Pending is the approval surface).
- Drag-resize columns / column-visibility toggles.
- Persistent filter state across tab close/reopen.
- Push-based reactivity when underlying data changes (user clicks **Refresh**).

## Architecture

Two new render-mode pages built on the same scaffolding v2.0 introduced for Review Pending and History:

1. `electron/renderer/dashboard-page.js` — replaces `output/dashboard.html`.
2. `electron/renderer/project-compare-page.js` — replaces `output/compare/<slug>.compare.html`.

One new backend command exposing existing compute as JSON:

3. `src/commands/compare-query.js` — `getProjectsSummary(db)` + `getProjectCompare(db, projectId)`. Both reuse `compareProject()` and `summarize()` from `src/compare.js`.

Two new IPC handlers + preload bridges:

4. `dlp:compare:summary` → `window.dlp.compare.summary()`
5. `dlp:compare:project` → `window.dlp.compare.project(projectId)`

One shared cell-rendering module so the compare table's DLD/SF buyer popups work without `<details>` (which needs an iframe document):

6. `electron/renderer/compare-cells.js` — native popup rendering for buyer/applicant cells.

No new database tables. No new migrations. No CLI changes beyond version bump.

## Data flow

```
[CLI compare]                                       [Renderer]
                                                          │
   ├─ writes output/dashboard.html                        │
   ├─ writes output/compare/<slug>.compare.html           │
   └─ populates pending_change + audit_log                │
                                                          ▼
                                  window.dlp.compare.summary() ──► main ──► getProjectsSummary(db)
                                  window.dlp.compare.project(id) ──► main ──► getProjectCompare(db, id)

Dashboard page    : calls summary() once on mount; renders cards grid.
Project Compare   : calls project(id) on mount; renders table.
Click card        : opens Project Compare tab.
Click row         : opens unit-history-panel side panel.
Click procedure # : dispatches `dlp:open-history` with { procedure_number } filter.
Click PENDING     : opens Review Pending tab with { unit_number } filter.
Refresh button    : re-runs the IPC call (cheap — no compare CLI rerun needed).
```

## § 1 — Backend query API

### `src/commands/compare-query.js`

```js
const { compareProject, summarize } = require('../compare');
const { listProjects } = require('./projects');

// One row per project. Heavy — calls compareProject() per project.
// Used by Dashboard mount.
function getProjectsSummary(db) {
  const rows = [];
  for (const p of listProjects(db)) {
    if (p.project_id == null) {
      // SF-only project — no DLD snapshot to compare. Surface as
      // empty card with source='SF only'.
      rows.push({
        project_id:     null,
        project_name:   p.project_name,
        source:         p.source,
        status:         'sf-only',
        total:          0,
        counts:         {},
        pending_count:  0,
        last_dld_at:    null,
        last_sf_at:     null
      });
      continue;
    }
    const result = compareProject(db, p.project_id);
    const counts = summarize(result.rows || []);
    const pending = db.prepare(
      `SELECT COUNT(*) AS n FROM pending_change
       WHERE project_id = ? AND decision = 'pending'`
    ).get(p.project_id).n;
    const dldSnap = db.prepare(
      `SELECT imported_at FROM dld_snapshot WHERE project_id = ?
       ORDER BY imported_at DESC LIMIT 1`
    ).get(p.project_id);
    const sfSnap = db.prepare(
      `SELECT imported_at FROM sf_snapshot ORDER BY imported_at DESC LIMIT 1`
    ).get();
    rows.push({
      project_id:    p.project_id,
      project_name:  p.project_name,
      source:        p.source,
      status:        result.status,
      total:         (result.rows || []).length,
      counts,
      pending_count: pending,
      last_dld_at:   dldSnap ? dldSnap.imported_at : null,
      last_sf_at:    sfSnap  ? sfSnap.imported_at  : null
    });
  }
  return rows;
}

// Full compare result for a single project. JSON-safe — same shape
// compareProject() already returns, ready for renderer consumption.
function getProjectCompare(db, projectId) {
  const result = compareProject(db, projectId);
  return {
    project:        result.project,
    status:         result.status,
    rows:           result.rows || [],
    counts:         summarize(result.rows || []),
    dldSnapshotId:  result.dldSnapshotId || null,
    sfSnapshotId:   result.sfSnapshotId  || null
  };
}

module.exports = { getProjectsSummary, getProjectCompare };
```

### IPC wiring (`electron/main.js`)

```js
ipcMain.handle('dlp:compare:summary', () => {
  const db = openDb();
  try { return getProjectsSummary(db); }
  finally { db.close(); }
});

ipcMain.handle('dlp:compare:project', (_evt, projectId) => {
  const db = openDb();
  try { return getProjectCompare(db, projectId); }
  finally { db.close(); }
});
```

### Preload bridge (`electron/preload.js`)

```js
compare: {
  summary: () => ipcRenderer.invoke('dlp:compare:summary'),
  project: (projectId) => ipcRenderer.invoke('dlp:compare:project', projectId)
}
```

## § 2 — Project Compare page

`electron/renderer/project-compare-page.js` exports `window.__renderProjectComparePage(container, projectId)`.

### Render contract

```js
function renderProjectComparePage(container, projectId) {
  container.classList.add('project-compare-page');
  container.innerHTML = SKELETON_HTML; // header + filter chips + search + table shell + refresh
  load();

  async function load() {
    showLoading();
    try {
      const data = await window.dlp.compare.project(projectId);
      renderHeader(data.project, data.counts, data.dldSnapshotId, data.sfSnapshotId);
      renderTable(data.rows, data.counts);
      wireInteractions();
    } catch (e) {
      showError(e);
    }
  }
}
```

Also surfaces (not new columns — overlays on existing ones):

- `sf_procedure_number` is rendered inside the SF Status cell as a clickable chip. Click → dispatches `dlp:open-history` with `{ procedure_number }`.
- `match_flags` includes `PENDING` when there are pending changes for that unit. Rendered as a `PENDING` chip in the Flags column. Click → dispatches a new `dlp:open-review-pending` custom event with `{ unit_number }`.

### Filter chips

Identical to current HTML: MATCH (off by default), PRICE ↑, PRICE ↓, BUYER, AREA, DLD-only, SF-only. Each is a toggle; counts in parens; click toggles. Same status colors as today.

### Search

Single text input. Matches against the combined cell text (lowercase) of every column — same as today's `data-search` attribute.

### Sort

Click column header → sort. Numeric-aware via `data-sort-val` (carry-over from current HTML). First click ascending; second descending; third toggles back.

### Row interactions

- **Row click** (anywhere except inside a popup cell or a chip) → opens unit-history-panel for `{ project_id, unit_number_norm: row.expected_sf_unit }`. `expected_sf_unit` is already the normalized unit identifier (output of `expectedSfUnit()` in `src/project-mapping.js`) and matches what `master_data.unit_number_norm` stores. Uses the existing `window.__openUnitHistoryPanel` introduced in v1.1.
- **DLD # / SF # cell** click → toggles a floating popup listing all buyers/applicants. See § 3.
- **SF Status cell** with a procedure number → click → `dlp:open-history`.
- **PENDING chip** click → `dlp:open-review-pending`.

### Refresh button

Top-right of the page. Reruns `window.dlp.compare.project(projectId)` and re-renders the table. No CLI invocation — `compareProject()` is pure query.

```js
function renderBuyerCell(container, buyers, kind /* 'dld' | 'sf' */) {
  // container is the <td>. Builds a chip + click handler that positions

  // closes. ESC closes. Single popup open at a time (close any other).

```

Visual parity with the HTML version: badge with count, names listed inside the popup with their secondary metadata (role / kind / amount / date). Reuses the same CSS classes (`buyer-popup`, etc.) by porting the relevant rules from `src/html-styles.js` into `styles.css`.

## § 5 — Sidebar + project picker wiring

### `electron/renderer/app.js`

Replace **two** url-mode opens with render-mode opens:

1. The `open-dashboard` action handler (line ~508):
   ```js
   // BEFORE
   const url = 'file:///' + currentDataFolder.replace(/\\/g, '/') + '/output/dashboard.html';
   window.__tabHost.open({ url, title: 'Dashboard' });

   // AFTER
   window.__tabHost.open({
     title: 'Dashboard',
     render: (container) => window.__renderDashboardPage(container)
   });
   ```

2. The `setProjectFilter` callback in `__initTopBar` (line ~102):
   ```js
   // BEFORE: opened file:///…/output/compare/<slug>.compare.html
   // AFTER:
   if (f && window.__tabHost && window.__renderProjectComparePage) {
     const pid = topBar.getProjectIdFor(f);   // small new helper on top-bar
     window.__tabHost.open({
       title: f,
       render: (container) => window.__renderProjectComparePage(container, pid)
     });
   }
   ```

   `getProjectIdFor(name)` is a new method exposed from `electron/renderer/top-bar.js`. The top-bar already fetches the full project list via `window.dlp.projects.list()` (one row per project with `project_id`, `project_name`, `source`); cache the list in a module-scope variable inside `refreshProjects()` and expose a synchronous lookup. For SF-only projects (`project_id` null) we fall back to opening the static HTML if it exists, otherwise show a not-applicable inline message inside the tab.

### Custom-event additions

`electron/renderer/app.js` adds an `dlp:open-review-pending` listener mirroring the `dlp:open-history` listener:

```js
document.addEventListener('dlp:open-review-pending', (ev) => {
  const filters = (ev && ev.detail) || {};
  window.__tabHost.open({
    title: 'Review pending',
    render: (container) => window.__renderReviewPendingPage(container, filters)
  });
});
```

This requires extending `__renderReviewPendingPage` to accept an initial-filter argument (currently it accepts none). The function already maintains its own filter state — we just seed it.

## § 10 — Risks & mitigations

| Risk | Mitigation |
|------|------------|
| `compareProject()` is heavy when called per project on Dashboard mount | Profile after Phase 1. If slow, add a lightweight `summarize-only` path that joins `pending_change` + `dld_snapshot` + `sf_snapshot` directly without running the full classifier. Deferred until measured. |
| CSS divergence between HTML and renderer | One-time port; both surfaces are stable. Document in `src/html-styles.js` header that renderer rules live in `styles.css`. |
| Popup z-index conflicts with tab strip / log column | Use a fixed-position div with high z-index, anchored via getBoundingClientRect at click time. |
| SF-only projects have no project_id | Dashboard renders them as empty cards; Project Compare opening for them falls back to legacy HTML (if it exists) or shows "not available" message. |
| Renderer tests don't exist | Manual smoke checklist gated on Phase 5 sanity audit; backend logic fully covered by Node tests. |
