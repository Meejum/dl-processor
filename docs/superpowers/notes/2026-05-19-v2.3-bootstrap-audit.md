## A. `pending_change` reads — compatibility with new `anomaly TEXT` column

Migration 009 adds `anomaly` as the LAST column, nullable, no default constraint, default NULL. Existing well-formed consumers (named columns or `SELECT *` into named bindings) are unaffected. Risk is positional binding (`SELECT *` consumed by index).

| File | Pattern | Risk |
|---|---|---|
| `src/pending-change.js` | named INSERT/UPDATE; `SELECT field, old_value, new_value …` named columns | None — explicit columns |
| `src/commands/review-pending.js` | named SELECT through `listPendingChanges()` helper | None |
| `src/commands/review-bps.js` | named SELECT joined with `sf_booking` / `dld_unit` | None |
| `src/commands/compare-query.js` | reads aggregates per project; doesn't read change rows | None |
| `src/compare.js` | calls `queueMasterDiffs(...)` — write path only | None |
| `src/compare-drift.js` | inserts only | None |
| `src/migrations/2026-05-12-003-pending-change-v2.js` | the table-creation migration | None — pre-009 |
| `electron/renderer/review-pending-page.js` | consumes IPC results which carry named fields | None |
| `electron/renderer/import-db-modal.js` | metadata query (counts only) | None |
| `src/db.js` | schema runner | None |
| `src/commands/db-backup.js` | full dump | None — `SELECT *` to backup is OK since destination is a parallel SQLite DB or JSON |

**Conclusion:** safe. No consumer makes positional assumptions about column order. The new `anomaly` column is invisible to every read site until v2.3 code starts populating + reading it.

## D. writeAuditLog signature

```js
writeAuditLog(db, {
  projectId, unitNumberNorm, tableName, field,
  oldValue, newValue, action, source, changeId, userNote,
  user, tier2
})
```

v2.3 rule firings will use `source: auditSourceFor.rule()` returning `'rule_fired'`, with `userNote` carrying `rule_id=<id>` for traceability. Bulk firings use `source: auditSourceFor.bulk()` returning `'bulk_op'`, with `userNote` carrying `batch=<UUID>`. Schema-light per spec § 6.3 — no new columns.
