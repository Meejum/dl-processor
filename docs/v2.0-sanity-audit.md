## Findings to defer to v2.1

1. **Retire the pre-Electron interactive CLI menu trio** — `src/menu.js`, `src/open-file.js`, `src/file-picker.js` are only referenced by each other and by their own tests. Safe to remove together along with `test/file-picker.test.js` + `test/open-file.test.js`. Saves ~600 LOC of dead surface.
2. **`teach-alias` scope dialog** — `review-pending-page.js` line 415-426 hard-codes `scope: 'project'` for v2.0. A modal picker for `global`/`project`/`unit` was punted from Task 15 per the plan; consider for v2.1.
3. **Open-in-SF deep link** — `bp-card.js` exposes the "Open in SF →" button but `review-pending-page.js` line 386-397 only copies the booking record id to the clipboard. A configurable SF instance URL pattern (e.g. `https://sobha.lightning.force.com/lightning/r/Booking__c/<id>/view`) belongs in Settings → v2.1.
4. **`window.parent.__openUnitHistoryPanel` comment in `unit-history-panel.js`** mentions iframe-hosted pages — true for the still-iframed Dashboard and compare-report tabs, but slightly outdated now that the inline pages call it directly. Comment could be tightened but is not actively misleading.
5. **`webSecurity: false` in `electron/main.js` line 38** — same v1.1 finding. The renderer reads its own generated output/ files via `<iframe src="file:///">`, which Chromium forbids without this flag. Replacing with a custom `file://` protocol handler scoped to the data folder is a defence-in-depth tightening for a later milestone.
6. **`__openImportDbModal` not loaded** error path in `app.js` line 542-544 logs to `console.error` only — staff would not see it in the in-app Errors pane. Minor; would only surface if the script load order in `index.html` was changed.
7. **Drift log tab tower/assigned filtering** — the v2.0 filter bar applies tower / assigned-to filters to Needs review (`applyClientSideBpFilters`), but the Drift log tab is filtered only by project / date / search (`visibleDriftRows`). Acceptable for v2.0 (drift rows do not carry SF context per the design); document this as a known limitation if anyone asks.

## Overall verdict

**v2.0 is ready to ship.** Every Feature C / Feature A / Feature B deliverable is wired end-to-end: snapshot-extract → compare-drift → CLI compare; migrations 006+007 → SF importer extracts step columns → review-bps groups + classifies → IPC handlers → preload bridge → bp-card component → review-pending-page filter bar. All 390 tests pass. CLI surface unchanged. Sidebar has no orphan buttons. The only inline fix needed was a five-line block-comment refresh in tab-host.js — code behaviour is unchanged. The seven follow-up items are all polish or defence-in-depth, none are correctness bugs. Recommend merging to master and tagging 2.0.0.
